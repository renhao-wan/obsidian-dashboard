# 笔记统计功能设计文档

## 概述

笔记统计功能是一个全新的功能模块，旨在为 Obsidian Dashboard 插件提供全面的笔记统计和分析能力。该功能将基于文件系统扫描，提供文件数量、文件大小、时间维度、内容分析等多维度的统计信息，并通过综合仪表盘展示。

## 设计目标

1. **全面性**：提供多维度的笔记统计功能
2. **性能**：支持大型 vault，响应迅速
3. **隐私**：所有数据本地处理，不上传到外部服务器
4. **实时性**：支持增量更新，反映最新变化
5. **可配置**：提供灵活的配置选项
6. **可扩展**：采用模块化架构，便于后续扩展

## 架构设计

### 整体架构

笔记统计功能采用**混合架构**，核心统计功能独立开发，UI 部分复用现有架构：

```
src/
├── sections/
│   └── stats/           # 统计功能模块
│       ├── index.ts     # 入口文件，导出渲染函数
│       ├── types.ts     # 类型定义
│       ├── scanner.ts   # 文件系统扫描器
│       ├── analyzer.ts  # 数据分析器
│       ├── cache.ts     # 缓存管理
│       └── views/       # 各统计视图
│           ├── overview.ts    # 概览视图
│           ├── timeline.ts    # 时间维度视图
│           ├── content.ts     # 内容分析视图
│           └── heatmap.ts     # 热力图视图
├── components/
│   └── stats/           # 统计专用组件
│       ├── charts.ts    # 自定义图表组件
│       ├── heatmap.ts   # 自定义热力图组件
│       └── tagcloud.ts  # 标签云组件
└── utils/
    └── stats/           # 统计工具函数
        ├── file-utils.ts    # 文件处理工具
        └── math-utils.ts    # 数学计算工具
```

### 模块职责

1. **scanner.ts**：负责扫描文件系统，提取文件元数据
2. **analyzer.ts**：负责分析数据，生成统计结果
3. **cache.ts**：负责缓存统计结果，提高性能
4. **views/**：负责渲染各统计视图
5. **components/**：负责渲染自定义图表、热力图、标签云

### 与现有架构的集成

- 复用现有的 section 渲染架构
- 复用现有的 UI 组件和样式
- 使用现有的回调机制
- 遵循现有的数据流模式

## 功能设计

### 第一阶段：基础统计功能

**功能描述**：实现基础的文件数量和大小统计

**具体功能**：
1. **概览卡片**
   - 总笔记数
   - 总文件大小
   - 今日新增笔记数
   - 本周新增笔记数

2. **文件类型分布**
   - 按文件类型统计数量
   - 按文件类型统计大小
   - 饼图或柱状图展示

3. **文件夹分布**
   - 按文件夹统计数量
   - 按文件夹统计大小
   - 树状图或柱状图展示

### 第二阶段：时间维度统计

**功能描述**：实现时间维度统计，如每日/每周/每月创建笔记数量

**具体功能**：
1. **创建时间分布**
   - 按日/周/月统计创建笔记数量
   - 折线图或柱状图展示

2. **修改时间分布**
   - 按日/周/月统计修改笔记数量
   - 折线图或柱状图展示

3. **时间趋势**
   - 最近 30 天/90 天/365 天的笔记创建趋势
   - 趋势图展示

### 第三阶段：内容分析功能

**功能描述**：实现内容分析功能，如标签云、关键词提取等

**具体功能**：
1. **标签统计**
   - 统计所有标签及其出现次数
   - 标签云展示

2. **关键词提取**
   - 从笔记内容中提取关键词
   - 关键词云展示

3. **笔记长度分布**
   - 统计笔记长度分布
   - 直方图展示

### 第四阶段：热力图展示

**功能描述**：实现热力图展示，类似 GitHub 贡献图

**具体功能**：
1. **创建时间热力图**
   - 按日统计笔记创建数量
   - 热力图展示

2. **修改时间热力图**
   - 按日统计笔记修改数量
   - 热力图展示

3. **自定义时间范围**
   - 支持选择时间范围
   - 支持自定义颜色方案

## 数据流设计

### 数据流概述

笔记统计功能的数据流遵循**单向数据流**模式：

```
文件系统 → Scanner → Analyzer → Cache → Views → UI
```

### 详细数据流

#### 1. 数据采集阶段
```
文件系统
    ↓
Scanner (文件系统扫描器)
    ↓
文件元数据列表
    ↓
Analyzer (数据分析器)
    ↓
统计结果
    ↓
Cache (缓存管理)
```

#### 2. 数据展示阶段
```
Cache (缓存管理)
    ↓
Views (各统计视图)
    ↓
UI 组件
    ↓
用户界面
```

### 增量更新机制

1. **文件系统事件监听**
   - 监听文件创建、修改、删除事件
   - 使用 Obsidian API 的 `vault.on()` 方法

2. **增量扫描**
   - 只扫描变化的文件
   - 更新缓存中的统计数据

3. **视图刷新**
   - 根据缓存变化刷新视图
   - 使用防抖机制避免频繁刷新

### 缓存策略

1. **内存缓存**
   - 统计结果缓存在内存中
   - 提高读取速度

2. **本地存储**
   - 缓存数据持久化到本地存储
   - 避免重复计算

3. **缓存失效**
   - 文件变化时自动失效
   - 定期检查缓存有效性

## 性能设计

### 性能目标

1. **大型 vault 支持**：支持 10,000+ 笔记的 vault
2. **快速响应**：统计结果在 1 秒内返回
3. **低内存占用**：内存占用控制在合理范围内
4. **流畅交互**：UI 交互流畅，无卡顿

### 性能优化策略

#### 1. Web Workers
- 将耗时的统计计算放到 Web Workers 中
- 避免阻塞 UI 线程
- 提高用户体验

#### 2. 增量计算
- 只计算变化的文件
- 避免重复计算
- 提高计算效率

#### 3. 虚拟滚动
- 对于大量数据的列表，使用虚拟滚动
- 只渲染可见区域的数据
- 减少 DOM 节点数量

#### 4. 懒加载
- 图表和热力图按需加载
- 减少初始加载时间
- 提高首屏渲染速度

#### 5. 防抖和节流
- 文件变化事件使用防抖
- UI 交互使用节流
- 避免频繁计算和渲染

### 内存管理

1. **及时释放**
   - 不使用的数据及时释放
   - 避免内存泄漏

2. **数据压缩**
   - 大量数据使用压缩存储
   - 减少内存占用

3. **缓存策略**
   - 使用 LRU 缓存策略
   - 自动清理过期数据

### 错误处理

1. **优雅降级**
   - 统计功能失败时不影响其他功能
   - 提供友好的错误提示

2. **重试机制**
   - 网络请求失败时自动重试
   - 文件扫描失败时跳过该文件

3. **日志记录**
   - 记录错误日志
   - 便于问题排查

## 配置设计

### 配置概述

笔记统计功能将提供**灵活的配置选项**，让用户可以根据自己的需求定制统计功能。

### 配置项

#### 1. 文件类型配置
```typescript
interface FileTypeConfig {
  enabled: boolean;
  extensions: string[];
  excludePatterns: string[];
}
```

**默认配置**：
- 启用：`.md`, `.txt`, `.org`
- 排除：`node_modules`, `.git`, `.obsidian`

#### 2. 统计指标配置
```typescript
interface StatsConfig {
  fileCount: boolean;
  fileSize: boolean;
  timeline: boolean;
  contentAnalysis: boolean;
  heatmap: boolean;
}
```

**默认配置**：全部启用

#### 3. 性能配置
```typescript
interface PerformanceConfig {
  useWebWorkers: boolean;
  cacheEnabled: boolean;
  cacheTTL: number;
  maxConcurrentScans: number;
}
```

**默认配置**：
- Web Workers：启用
- 缓存：启用
- 缓存 TTL：5 分钟
- 最大并发扫描数：4

#### 4. UI 配置
```typescript
interface UIConfig {
  defaultView: 'overview' | 'timeline' | 'content' | 'heatmap';
  theme: 'auto' | 'light' | 'dark';
  chartLibrary: 'custom' | 'chartjs';
}
```

**默认配置**：
- 默认视图：概览
- 主题：自动
- 图表库：自定义
